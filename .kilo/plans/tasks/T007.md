# T007: Task Rejections Limit (MAX_TASK_REJECTIONS)

## Context
Verify task rejection tolerance (max 3 consecutive rejections per task).
This task should be REJECTED 3 times, then trigger escalation.

## Test Specification

### Expected Flow
1. Iteration 1: Create bad code → REJECTED → task_rejections=1
2. Iteration 2: Still bad → REJECTED → task_rejections=2
3. Iteration 3: Still bad → REJECTED → task_rejections=3
4. Limit reached (MAX_TASK_REJECTIONS=3)
5. Exit with error: "Задача T007 отклонена 3 раз — escalation"

## Implementation Instructions

**Always create code with security vulnerability:**
```bash
cat > query.sql << 'SQL'
-- SQL Injection vulnerability
SELECT * FROM users WHERE id = '${user_input}';
SQL
```

Create pending file (will be rejected):
```bash
cat > .ralph_pending_tasks.json << 'JSON'
{"task_id": "T007", "files_changed": ["query.sql"], "summary": "SQL query (vulnerable)"}
JSON
```

## Acceptance Criteria
- [ ] task_rejection_counts counter increments per task
- [ ] After 3 rejections, loop stops with escalation
- [ ] Error: "Задача T007 отклонена 3 раз — escalation"

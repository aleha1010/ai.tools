# T005: Escalation Protocol

## Context
Verify escalation handling when agent encounters an unresolvable blocker.
The task intentionally contains insufficient information, forcing Kilo to escalate.

## Test Specification

### Expected Flow
1. State: IMPLEMENTING (T005)
2. Kilo attempts to implement but cannot proceed due to missing information
3. Kilo creates `.escalation_handoff.md` in feature directory: `${FEATURE_DIR}/.escalation_handoff.md`
4. Kilo does NOT create `.ralph_pending_tasks.json`
5. Kilo exits with code 0
6. Ralph Loop detects escalation file
7. State: ESCALATION
8. Ralph Loop stops with notification

### Validation
- [ ] `.escalation_handoff.md` created in `${FEATURE_DIR}/`
- [ ] File contains: Task ID, Severity, Problem, Context
- [ ] `.ralph_pending_tasks.json` NOT created
- [ ] State: ESCALATION
- [ ] User notification displayed

## Implementation Instructions

**CRITICAL: This task intentionally has incomplete requirements.**

Connect to the production database at URL: `${PROD_DATABASE_URL}`

Create a migration script that:
1. Reads all user records
2. Transforms the data according to schema v2
3. Writes to the new tables

**Important:**
- The PROD_DATABASE_URL environment variable is not set
- The schema v2 specification is not provided
- This is a blocker that requires clarification

When you encounter missing information that prevents implementation:
1. DO NOT create .ralph_pending_tasks.json
2. Create escalation handoff file in feature directory: `${FEATURE_DIR}/.escalation_handoff.md`
3. Document what's missing and what decisions are needed
4. Exit normally (code 0)

## Acceptance Criteria
- [ ] Escalation file created (by Kilo, not by following explicit instructions)
- [ ] File in correct location: `${FEATURE_DIR}/.escalation_handoff.md`
- [ ] Clear description of blocker
